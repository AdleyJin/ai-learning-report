# 学情报告 & 自定义能力 — 页面设计 Brief

> 说明：基于产品 PRD《学情报告 & 自定义能力》输出的页面设计 Brief，不含视觉方案；组件优先采用 HeroUI v3。
> 当前流程：选择模版 → 输入分析内容 → 提交 → Agent 解析 → 展示报告 → 查看 / 下载。

---

## 1. 页面目标

- 在「自由提问」之外，提供**标准化的学情可视化报告**这一交付物形态。
- 让用户通过**选择预设模版 + 输入分析内容**快速获得标准化可视化报告，无需自行维护模版。
- 让用户顺畅完成**提交 → 等待解析 → 查看 / 下载**的闭环。
- 通过顶部栏**报告中心**入口，让用户在对话流之外**集中浏览、回看**历史报告。

## 2. 用户任务

- 选择报告模版。
- 输入分析内容（分析对象、时间范围等）并提交。
- 等待 Agent 异步解析并生成报告。
- 查看报告全文。
- 下载报告。
- 打开报告中心，浏览历史报告列表并进入任意一份查看详情。

## 3. 信息架构

```
AI 学情主界面
├── 顶部栏（head）
│   ├── 产品标识
│   └── 报告中心入口（右侧）
├── 对话流区
│   ├── Agent 回答
│   └── 学情报告卡片（解析中 / 已完成；含下载入口）
└── 输入区
    └── 模版选择 + 分析内容输入 + 提交

报告中心侧栏（右侧侧栏 · 与报告详情共用容器）
├── 列表态
│   ├── 标题栏（标题「报告中心」+ 关闭；可含搜索 / 筛选）
│   └── 报告列表（每项：标题 / 时间 / 格式 / 状态）
└── 详情态（由列表项或报告卡片进入）
    ├── 标题栏（返回列表 + 下载入口）
    └── 报告正文（内容模块 / 指标 / 图表，Tabs 切换）

学情报告详情（ClassIn 内弹窗 / 侧边栏）
├── 标题栏（含下载入口）
└── 报告正文（内容模块 / 指标 / 图表，Tabs 切换）
```

## 4. 用户与 Agent 交互流程

```
用户：进入主界面 → 选择模版 → 填写分析内容 → 提交
                                        │ (报告卡片进入解析中 loading，前端轮询)
Agent：解析 query → 识别报告需求 → 规划报告内容 → 调用工具(数据获取/指标计算/前端渲染) → 组装返回报告
        │                                 │
        └────────── 任一环节失败 ───────────┘ → 解析/生成失败（重试回提交）
                                        │
用户：报告卡片(已完成) ├→ 查看 → 报告详情(Modal/Drawer · Tabs) → 下载
                      └→ 下载 → 系统下载

用户：顶部栏报告中心入口 → 展开侧栏(列表态) → 浏览历史报告列表
                                  │ (列表加载中 / 空 / 失败三态)
                                  └→ 点击已完成列表项 → 报告详情(详情态) ⇄ 返回列表 → 下载
```

## 5. 页面模块

| 模块 | 职责 |
| --- | --- |
| 模版选择 | 从预设模版中选择一个 |
| 分析内容输入 | 填写分析对象、时间范围等并提交 |
| 对话流 | 承载 Agent 回答与学情报告卡片 |
| 学情报告卡片 | 解析中（loading）与已完成（静态入口 + 下载）两态 |
| Agent 处理反馈 | 解析中进度反馈、失败提示与重试 |
| 学情报告详情 | 弹窗 / 侧边栏展示报告全文 |
| 下载能力 | 报告卡片按钮 + 报告详情标题栏按钮 |
| 报告中心入口 | 顶部栏右侧入口，点击展开侧栏列表态 |
| 报告列表 | 侧栏内集中展示历史报告，按时间倒序；支持点击进入详情 |
| 报告列表项 | 标题 / 生成时间 / 格式 / 状态；仅已完成可点击 |
| 列表 ⇄ 详情导航 | 列表项进入详情、详情返回列表的切换 |

## 6. 组件建议（优先 HeroUI）

| 场景 | HeroUI 组件 | 说明 |
| --- | --- | --- |
| 模版选择 | `Select` / `Combo Box` | 单选预设模版 |
| 分析对象输入 | `Combo Box` / `Autocomplete` / `Tag Group` | 支持搜索与多选 |
| 时间范围 | `Date Range Picker` | 填充时间区间 |
| 提交 / 下载按钮 | `Button` / `Button Group` | 主操作与按钮区 |
| 输入区组织 | `Form` + `Text Field` / `Text Area` | 内容输入与校验 |
| 报告卡片 | `Card` | 卡片入口 |
| 解析中状态 | `Skeleton` / `Spinner` / `Progress Circle` | 异步解析 loading |
| 报告详情容器 | `Modal`（弹窗）/ `Drawer`（侧边栏） | 二选一或按场景并存 |
| 详情内导航 | `Tabs` | 报告多模块切换 |
| 文件类型选择 | `Dropdown` / `Select` | html / pdf / word |
| 结果反馈 | `Toast` / `Alert` | 下载成功、失败提示 |
| 失败 / 权限提示 | `Alert` / `Alert Dialog` | 错误与权限不足 |
| 辅助说明 | `Tooltip` / `Chip` / `Badge` | 状态标识与提示 |
| 报告中心入口 | `Button`（icon + 文字）+ `Badge` | 顶部栏入口，Badge 显示报告数量 |
| 报告列表 | `List Box` / `Card` 列表 | 集中展示历史报告项 |
| 列表搜索 | `Search Field` | 按标题检索报告（可选） |
| 列表筛选 | `Tabs` / `Select` / `Tag Group` | 按状态 / 模版类型筛选（可选） |
| 列表项状态 | `Chip` | 已完成 / 解析中 / 失败标识 |
| 返回列表 | `Button`（icon-only，返回箭头） | 详情态返回列表入口 |
| 列表加载态 | `Skeleton` / `Spinner` | 列表拉取占位 |

## 7. 页面状态

| 状态 | 表现 |
| --- | --- |
| 默认 | 提供模版选择与内容输入；已完成报告卡片为静态可点击入口（含下载） |
| 加载中 | 提交后 Agent 异步解析，卡片展示 loading 动效；前端轮询直至「生成完毕」后转静态入口 |
| 空状态 | 暂无报告时的占位与引导（PRD 未展开，需确认） |
| 错误 | 解析失败 / 下载失败的提示与**重试**（PRD 未展开，需确认） |
| 权限不足 | 无权查看 / 下载时的拦截提示（PRD 未展开，需确认） |
| 报告中心 · 列表加载中 | 拉取历史报告列表时的占位（Skeleton / Spinner） |
| 报告中心 · 列表为空 | 尚无任何报告时的占位与引导（引导去生成第一份报告） |
| 报告中心 · 列表错误 | 列表拉取失败的提示与重试 |

## 8. 交互规则

- 报告为**异步生成**：提交后先出解析中卡片，前端轮询后端状态，完成后切换为可点击的静态入口；**未完成前不提供「查看全文」可点击入口**，避免进入空白 / 报错。
- **解析失败可重试**：失败态需明确提示，并提供重新提交的路径（回到提交步骤）。
- 报告详情在 **ClassIn 内**以弹窗 / 侧边栏打开，多模块通过 `Tabs` 切换。
- **下载覆盖所有交付物形态**（对话结果、学情报告等），统一下载为 PDF；下载入口两处：报告卡片、报告详情标题栏。
- 点击下载唤起**系统下载**。
- 用户直接选择预设模版，**不涉及模版的维护 / 新建 / 编辑**。
- **报告中心与报告详情共用右侧侧栏容器**：顶部栏入口进入「列表态」，列表项 / 报告卡片进入「详情态」，两态可来回切换。
- 报告列表**按生成时间倒序**；**仅「已完成」列表项可点击进入详情**，解析中 / 失败项仅以状态标识呈现。
- 从报告中心进入详情时必须提供「**返回列表**」入口；返回后回到列表态的浏览位置。

## 9. 设计风险

- **Agent 解析过程不透明**：解析→识别→规划→调用工具多步骤耗时不确定，若仅有单一 loading 易让用户焦虑；建议考虑是否暴露阶段性进度反馈。
- **异步等待与超时兜底**：轮询最大次数、超时提示、失败重试文案 PRD 未定义，需补齐。
- **空 / 错误 / 权限三态缺失**：PRD 未展开，落地前必须补齐，否则边界场景无设计可依。
- **弹窗 vs 侧边栏未定**：报告详情形态待设计侧确认，影响信息密度与多模块导航方案。
- **文件类型未收敛**：html/pdf/word「更多待定」，下载格式扩展性需在交互上预留。
- **ClassIn 容器约束**：嵌入 ClassIn 的尺寸 / 滚动 / 层级限制可能影响弹窗与侧边栏可用空间。
- **报告中心与详情共用侧栏的态切换**：列表态 ⇄ 详情态切换需保持清晰的导航与位置记忆，避免用户迷失；移动端侧栏为全屏时尤其要保证返回路径明确。
- **列表规模与性能**：历史报告数量增长后，列表需考虑分页 / 虚拟滚动 / 搜索筛选，PRD 未定义加载策略，需补齐。
- **入口可发现性**：报告中心入口置于顶部栏，需与产品标识、其他全局操作的视觉权重协调，避免被忽略或过度抢眼。
